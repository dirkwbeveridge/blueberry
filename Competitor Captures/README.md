# Competitor Capture Workflow

This folder stores primary evidence for competitor intelligence.

## Folder Structure

- Huckleberry/
- Sprout-Pregnancy-3D/
- Sprout-Baby-Tracker/
- Nara/
- CAPTURE-LOG.md

## Naming Convention

Use this filename format for all screenshots:

YYYY-MM-DD_app-flow_screen-seq_source.png

Examples:

- 2026-06-21_huckleberry-onboarding_01_appstore.png
- 2026-06-21_nara-tracking-home_02_inapp.png

## Required Metadata Per Capture

Every screenshot set must have a `metadata.md` file in the app folder with:

- Capture date and timezone
- Source URL or app version
- Device/context (App Store web, iPhone simulator, physical iPhone)
- Capture type (appstore listing, in-app walkthrough)
- Notes on visual style and UX patterns

## Evidence Rules

- Do not include claims without source URLs.
- Keep one section per flow: onboarding, home, logging, insights, collaboration, premium/paywall, settings.
- Mark uncertain claims as "Unverified".

## Legal and Usage

These artifacts are internal research references only.
Do not redistribute competitor screenshots externally.
