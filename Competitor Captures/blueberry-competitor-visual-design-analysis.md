# Competitor UI Design Analysis
*Based on App Store screenshots — June 2026*

---

## Huckleberry

### Screen: Home / Daily Log
- **Layout:** Card-based grid. Activity types (sleep, feed, diaper) shown as distinct tiles in a warm, cream-toned home screen
- **Color system:** Warm orange-amber primary (#E8974A range), creamy off-white background, soft teal accents for secondary actions
- **Typography:** Rounded sans-serif throughout. Display weights for time values (large, bold). Body in light weight
- **Data density:** Low to medium — shows "last logged" callouts per category with simple icons. Not overwhelming
- **Tone:** Friendly, warm, slightly playful. Illustrated icons, not photographic

### Screen: SweetSpot / Sleep Prediction
- **Visual:** Clock-style UI with color-coded "window" highlighting optimal nap timing. Uses a circular/arc format
- **Color signal:** Green = good, yellow = caution, red = overdue. Traffic-light language
- **Key insight:** The entire screen is built around one number (the prediction time). Everything else steps back. Contrast with Blueberry's home which has many competing cards

### Screen: Berry AI Chat
- **Layout:** Standard chat bubble interface. Lavender/purple bubbles for Berry, white for user
- **Brand signal:** Berry's avatar is a simple round character in Huckleberry's orange brand. The purple chat bubbles are notably close to Blueberry's color system
- **Context display:** Small "Week X · Baby's age" shown at top of chat — the most visible context injection they do
- **Prompt suggestions:** Horizontal chip row above keyboard. Common questions visible before user types
- **Key weakness visible in screenshot:** No indication of partner context. It's a single-user chat. No "Marina + Dirk" framing anywhere

### Screen: Widgets (iOS Home Screen)
- **Multiple widget sizes:** Small (single metric), medium (2-up activity), large (timeline)
- **Key insight:** Widget offering is a major product investment — shows depth of commitment to iOS native experience. Blueberry doesn't have this yet but should plan for it

### Huckleberry Design Language Summary
| Attribute | Assessment |
|---|---|
| Color temperature | Warm (amber/orange dominates) |
| Visual complexity | Low — very focused, one thing per screen |
| Information hierarchy | Strong — big number, supporting context, actions below |
| Tone | Friendly, competent, not clinical |
| Mobile-native quality | High — clearly built iOS-first |
| Partner awareness in UI | Zero — single-user experience visually |

## Sprout Pregnancy Tracker 3D

### Screen: 3D Baby Model View
- **The signature feature:** Photorealistic 3D render of the baby at the current week, shown in a dark background that makes the baby "float" — cinematic presentation
- **Animation:** Baby moves subtly. Heartbeat is audible. This is the most technically impressive visual in the entire category
- **Color system:** Dark background for the baby view, contrasting sharply with the lighter content screens. Two visual modes in one app
- **Typography:** Clean sans-serif, medical-adjacent. No personality font — functional

### Screen: Week Timeline
- **Layout:** Linear timeline scroll with week markers. Milestone dates (first OB visit, anatomy scan, etc.) shown as flags on the timeline
- **Color system:** Teal/green (#6DB3A8 range) primary. Medical professional aesthetic. Clean but cold
- **Information density:** High compared to Huckleberry. More text content per screen
- **Journal screen:** Photo grid with timestamps. Export to ebook badge visible — clear keepsake positioning

### Screen: Widgets + Apple Watch
- **Similar to Huckleberry:** Widget offering with multiple sizes. Shows kick counter on Apple Watch — strong use of wearable
- **Key insight:** Even a solo-founder-scale app (Med ART Studios) has deep iOS integration including Watch and Widgets. Table stakes for launch

### Sprout Design Language Summary
| Attribute | Assessment |
|---|---|
| Color temperature | Cool-neutral (teal/medical green) |
| Visual complexity | Medium — more information per screen |
| Signature moment | 3D baby render (unique in category) |
| Tone | Trustworthy, physician-adjacent, slightly clinical |
| Mobile-native quality | High — Watch, Widgets, timeline |
| Partner awareness in UI | None — zero partner UI visible anywhere |
| Emotional warmth | Low — designed to inform, not connect |

## Nara Baby Tracker

### Screen: Home / Activity Dashboard
- **Layout:** Clean vertical stack. Large, tappable cards for each category (feed, sleep, diaper, etc.) with last-logged timestamp prominent
- **Color system:** Near-white background, very light neutrals, with a single teal/mint accent for active states. Extremely restrained
- **Typography:** Large body text — the highest legibility of the three apps. Clearly designed for 4 AM use. No decorative font choices
- **Tone:** Calm, minimal, trustworthy. Closest to iOS native design language

### Screen: Sleep Timer
- **Visual:** Full-screen timer with large digit display. Start/stop is the entire interaction
- **Color signal:** Background shifts to a dark/calm state when tracking is active
- **Key insight:** Single-purpose, distraction-free. The anti-Huckleberry in terms of density

### Screen: Activity Log / Stats
- **Graphs:** Bar and line charts for daily activity patterns. Clean, no visual noise
- **Color:** Consistent teal accent. Charts are minimal, not elaborate

### Screen: Growth Charts
- **Visual:** WHO-percentile growth curves shown with the baby's plotted measurements. Professional, easy to share with pediatrician
- **Design signal:** This is clearly designed to print or share — clinical utility over aesthetics

### Screen: Multi-Caregiver
- **Visual:** Simple "Add caregiver" modal with invite mechanism. No role differentiation visible — everyone sees the same home screen
- **Key gap visible in screenshot:** The caregiver view is identical to the primary user view. No partner-specific lens

### Nara Design Language Summary
| Attribute | Assessment |
|---|---|
| Color temperature | Cool-neutral (mint/teal, near-white) |
| Visual complexity | Very low — the most minimal of the three |
| Readability at 4 AM | Highest of all three |
| Tone | Calm, functional, welcoming |
| Mobile-native quality | High — Live Activities, Dynamic Island, Watch |
| Partner awareness in UI | Caregiver sync only — same UI, no role differentiation |
| Emotional warmth | Low-medium — clean, not intimate |

## Blueberry Design Opportunities from Screenshots

1. **Warmth as differentiation:** Every competitor skews cool (teal, medical, white). Blueberry's deep purple + lavender + warm off-white genuinely stands out on the App Store shelf.

2. **Serif typography is unmistakably different:** No competitor uses a display serif. Playfair Display on week headings and card titles creates an editorial, journalistic feel — closer to a beautiful pregnancy book than a health app.

3. **Partner view is invisible in all three apps.** There is no screenshot in any of their App Store pages showing a partner-specific experience. If Blueberry takes a screenshot of the Partner tab with the activity feed and suggestion card — that screenshot does not exist in this category. That's the App Store screenshot we lead with.

4. **Berry's AI chat uses purple bubbles.** Coincidentally close to Blueberry's palette. If we launch with our AI chat, we need to differentiate visually — consider a soft blueberry-tinted gradient bubble rather than solid purple, or an AI avatar using the 🫐 motif they'd already recognize.

5. **Widget gaps are real but deprioritized:** All three have widgets. Blueberry should plan for them in Phase 2 — especially a lock screen widget showing "Week 15 · 176 days to go" in brand type.
