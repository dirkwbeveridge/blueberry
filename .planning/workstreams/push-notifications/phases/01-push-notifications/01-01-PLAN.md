## Guardrails

**PRIVACY_BOUNDARY (non-negotiable product promise):** Push payloads must never contain raw health-log fields: `symptoms`, `mood`, `energy_level`, `notes` (raw), or `weight_kg`. Permitted payload body: a shared sentence Mom explicitly chose to share (`shared_note`), an appointment title, a todo title, or a generic signal string ("She logged a check-in"). This is enforced server-side by `assertNoHealthData` in `supabase/functions/shared/pushPayload.ts` and must be called before every push dispatch.

**No Expo Push Service.** This project dispatches directly to APNs and FCM. No request to `exp.host` should appear anywhere. On the client, call `getDevicePushTokenAsync()` (native token) not `getExpoPushTokenAsync()` (Expo token). The difference: `getDevicePushTokenAsync` returns the raw APNs device token or FCM registration token. `getExpoPushTokenAsync` routes through Expo's servers — do not use it.

**No service-role key on the client.** The Supabase service-role key exists only inside Edge Functions as the `SUPABASE_SERVICE_ROLE_KEY` environment variable. Never reference it in any file under `app/`, `hooks/`, `lib/`, `store/`, or `components/`.

**Quiet hours are server-side.** The Edge Functions check quiet hours before dispatching. Client-side quiet-hours logic is additive UX polish, not the enforcement layer.

**No EAS required.** There is no `projectId` under `extra.eas` in `app.json`. Do not add one. `getDevicePushTokenAsync()` does not require a projectId.

## Required Secrets

The following must be configured in the Supabase Dashboard under Project Settings → Edge Functions → Secrets before any Edge Function can dispatch. Document each in `SUPABASE-SETUP.md`.

**APNs (iOS):**
- `APNS_AUTH_KEY` — full PEM content of the `.p8` auth key downloaded from Apple Developer Portal (Key Services → Keys). Paste the entire file including `-----BEGIN PRIVATE KEY-----` and `-----END PRIVATE KEY-----`.
- `APNS_KEY_ID` — 10-character Key ID shown in Apple Developer Portal next to the key.
- `APNS_TEAM_ID` — 10-character Apple Team ID (visible in developer.apple.com top-right).
- `APNS_BUNDLE_ID` — the app bundle identifier, e.g. `com.blueberry.app`.
- `APNS_ENV` — `sandbox` for development builds, `production` for App Store / TestFlight. Set to `sandbox` initially.

**FCM (Android):**
- `FCM_SERVICE_ACCOUNT_JSON` — full contents of the Firebase service account JSON file (Firebase Console → Project Settings → Service Accounts → Generate new private key).
- `FCM_PROJECT_ID` — Firebase project ID (visible in the service account JSON as `project_id`).

**Supabase (all functions):**
- `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` — auto-injected by Supabase into every Edge Function; no manual configuration needed.

## Schema Migration

Run in Supabase SQL Editor after reviewing:

```sql
-- Phase 7c: Push Notifications (native APNs/FCM)
-- Adds push token storage, platform flag, and notification preferences to users.
-- Adds share_with_partner flag to health_logs for opt-in partner check-in events.

ALTER TABLE users
  ADD COLUMN IF NOT EXISTS push_token               text,
  ADD COLUMN IF NOT EXISTS push_platform            text CHECK (push_platform IN ('ios', 'android')),
  ADD COLUMN IF NOT EXISTS notif_appointments       boolean NOT NULL DEFAULT true,
  ADD COLUMN IF NOT EXISTS notif_partner_checkin    boolean NOT NULL DEFAULT true,
  ADD COLUMN IF NOT EXISTS notif_todo_assigned      boolean NOT NULL DEFAULT true,
  ADD COLUMN IF NOT EXISTS notif_kick_reminder      boolean NOT NULL DEFAULT false,
  ADD COLUMN IF NOT EXISTS quiet_start              int     NOT NULL DEFAULT 21,
  ADD COLUMN IF NOT EXISTS quiet_end                int     NOT NULL DEFAULT 7;
-- push_platform: 'ios' → APNs dispatch; 'android' → FCM dispatch.
-- quiet_start / quiet_end are local-time hours (0-23). Default: 9 pm to 7 am.

ALTER TABLE health_logs
  ADD COLUMN IF NOT EXISTS share_with_partner  boolean NOT NULL DEFAULT false,
  ADD COLUMN IF NOT EXISTS shared_note         text;
-- shared_note is the ONLY health-log field permitted in a push payload.
-- Raw symptom/mood/energy/notes fields remain off-limits even when share_with_partner is true.

-- Index: Edge Function queries partner token by household lookup
CREATE INDEX IF NOT EXISTS idx_users_push_token ON users(household_id) WHERE push_token IS NOT NULL;
```

Also update `supabase-schema.sql` (the canonical schema file) to include these columns.

## Dependency Note

**Simulator limitation:** APNs device tokens cannot be obtained on iOS Simulator. `registerForPushNotifications` guards this via `expo-device`'s `isDevice` check. **Apple Developer Program:** APNs Auth Key (p8) requires an Apple Developer Program membership. The key is created at developer.apple.com → Certificates, Identifiers & Profiles → Keys → Add (+) → Enable Apple Push Notifications service (APNs). Download the key once — it cannot be re-downloaded.
